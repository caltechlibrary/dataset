Windows Notes

Windows’ command prompt presents some challenges for working with JSON on the command line. A command line 

    dataset create T1.ds one '{"one":1}'

which would work in a POSIX shell fails. The command prompt makes the JSON expression look like {one:1} which is NOT JSON and also not a filename. As a result working with dataset at the Windows command prompt requires conforming to the command prompt’s expectation on quoting. This will work. 

    dataset create T1.ds one "{"""one""":1}
